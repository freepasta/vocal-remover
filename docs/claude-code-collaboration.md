# Claude Code 协作开发实践总结

本文总结了本次开发过程中，**人和 AI 协作编程**的经验。不是代码技巧，而是怎么和 Claude Code 有效对话。

---

## 1. 需求表达：先整体后细节，逐步迭代

### 好的实践 ❗

**第一步先说整体目标**：
> "创建一个能够去除音乐中人声、只保留伴奏的程序，输入支持MP3格式，添加图形用户界面"

这样 Claude 能把握整体方向，然后一步步拆解。

**第二步逐步提具体要求**：
> "你调整一下代码，除了输出wav格式外，也输出mp3格式。"
> "能设计一个交互界面吗？不要用命令行。"
> "你看一下怎么没有开始处理的按钮呢？"
> "还是不行，必须最大化才看得见，把按钮调整到日志上面吧"

每次只解决一个问题，逐步逼近最终结果。

### 不好的做法

不要一次性扔出几十行非常细碎的要求，AI 容易漏掉一些。
也不要什么都不说就让"你看着办"，AI 不知道你的偏好。

---

## 2. 问题反馈：直接说现象，给截图

### 好的实践

**直接说看到了什么问题**：
> "你看一下怎么没有开始处理的按钮呢？"
> "还是不行，我必须把窗口最大化才显示的出来。"
> "报错了 [截图]"
> "github上面，我看README里面英文版里图片显示不正常，你用这张图吧"

**有错误截图一定要给截图**，比文字描述清楚得多。

### 不好的做法

不要说"代码不对，你自己找bug"，你说清楚现象定位更快。

---

## 3. 代码修改：让 AI 自己读文件，不用你粘贴

### 好的实践

直接告诉文件路径就行：
> "你看一下 gui.py 按钮位置对不对"
> "检查一下中英文README内容一致吗"

Claude Code 能直接读取你的本地文件，不需要你复制粘贴代码。

### 不好的做法

不用你手动把代码复制过来，浪费你的时间。

---

## 4. 测试验证：让 AI 自己运行测试

### 好的实践

> "我在 test_input 添加了 司南 - 苏州人家_502611916.mp3，请用这个文件做一下测试。"
> "你自己到网上下载一个音乐，测试一下对比效果。"

让 AI 在你环境上直接运行，能发现环境相关的问题（ffmpeg路径、编码问题等）。

---

## 5. 项目收尾：让 AI 整理文档

### 好的实践

> "总结一下这次对话，整理开发回顾到一个markdown文件"
> "从对话中总结一下和Claude协作的最佳实践"

AI 很擅长整理文档，能帮你把过程记录下来，方便后面分享给其他人。

---

## 6. 本次项目的协作节奏

```
需求提出 → 写出核心代码 → 测试 → 反馈问题 → AI修复 → 再测试 → 再修复 → ... → 文档整理 → 收尾
   ↓
完成核心功能 → 加GUI → GUI布局问题 → 调整布局 → 中文编码崩溃 → 修复编码 → README同步 → 添加截图 → 总结文档
```

**整个过程一共不到一个小时**，就从0到1完成了一个带GUI的完整应用。

---

## 7. 给使用者的建议

1. **放心让 AI 写代码**，出了问题再调整，比你自己写更快
2. **一次只改一个问题**，改完测试，没问题再下一个
3. **布局/UI问题**，直接截图说效果，比文字描述清楚
4. **编码/依赖环境问题**，让 AI 自己在你机器上跑，能自己发现问题
5. **最后让 AI 整理文档**，省得你自己写，出来直接能用

---

## 8. 关键心法

> **AI是你的结对编程伙伴，不是魔法盒子**

- 你告诉它"做什么"和"哪里不对"
- 它帮你写代码、调试、整理文档
- 有问题就直接说，它会不断修改直到对为止
